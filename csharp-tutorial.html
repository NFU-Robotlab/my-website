一、硬體連線配置

如何透過通訊轉換器與電源轉接板正確連接馬達

要讓電腦直接控制馬達，必須先準備專屬的電源轉接系統與訊號通訊轉換器。請集齊以下必備硬體設備：

🛠️ 實驗室必備通訊硬體：

  1. 12V 電源供應器：為馬達提供足夠的動力。
  2. AX-12(A) 伺服馬達：實驗室的核心關節作動器。
  3. 通訊轉換器（USB to Serial）：將電腦的 USB 信號轉換為 TTL 或 RS-485 訊號。
  4. 電源轉接板：將馬達與 12V 供電端和訊號端並聯橋接。
  5. 2 條 3-Pin 連接線。
▲ 必要控制設備與通訊線材

★ 實機接線與通訊協定開關設定

將設備如下圖所示連接。在通電前，請務必先撥動通訊轉換器上的模式指撥開關：
• TTL 模式：適用於 AX-12(A) / MX 等 3-Pin 系列馬達（本實驗項目採用此設定）。
• RS-485 模式：適用於 DX / RX / EX 等高階 4-Pin 系列馬達。
⚠️ 警告：若指撥切換錯誤，電腦將完全讀取不到馬達訊號，甚或造成硬體衝突。

▲ 電源與通訊硬體串接總覽

✓ 使用 Dynamixel Wizard 驗證 COM 埠與鮑率

1. 完成連線後，開啟 RoboPlus 主程式，點選上方工具列選單切換至 「Expert」模式，接著點選 「Dynamixel Wizard」 工具。
2. 在 Wizard 視窗中選取通訊轉換器的 COM Port（可在 Windows 「裝置管理員 -> 連接埠」中查看），點選 「Start searching」 開始自動搜尋。

▲ Dynamixel Wizard 掃描與偵錯軟體
▲ 掃描成功：獲取馬達實際 ID 與鮑率參數

⚠️ 請確實記錄馬達的 ID（預設為 1）與傳輸鮑率（預設為 1 Mbps / 57600 bps 等），這將在接下來的 C# SerialPort 設定中扮演決定性關鍵！

🛡️ Checksum (校驗和) 算法原理：

算法公式：Checksum = 0xFF - (ID + Length + Instruction + Parameter 1 + ... + Parameter N)

目前計算過程：0xFF - (01 + 07 + 03 + 1E + 00 + 02 + 00 + 02) = 0xFF - 31 = BF

二、C# 軟體開發與 SerialPort 封包傳輸

如何在 Visual Studio WinForms 中設置 SerialPort 發送控制封包

實驗室最推薦使用免費且穩定的 Visual Studio Community 版本 進行 Windows Form 應用程式開發。以下是完整的 C# 專案序列通訊配置指南：

1

在 UI 設計中加入 SerialPort 元件

建立「Windows Forms App (.NET Framework)」專案。在 Form1.cs [Design] 頁面中，打開側邊的 Toolbox（工具箱），找到 SerialPort 元件並直接雙擊拖拉到畫面上。接著在右下角的 Properties（屬性） 視窗，設定對應的 PortName 與 BaudRate。

▲ 於 WinForms 設計檢視中拖入 SerialPort 元件
2

發送控制封包範例代碼 (C#)

雙擊 Form 元件進入程式編輯，將以下 C# 發送函數複製到你的專案類別內。此方法會完全遵循 Protocol 1.0 的通訊格式： 

/// <summary>
/// 控制 AX-12(A) 馬達的目標位置與運動速度
/// </summary>
/// <param name="id">馬達通訊 ID (1-253)</param>
/// <param name="position">目標角度位置 (0-1023)</param>
/// <param name="speed">運作扭矩速度 (0-1023)</param>
private void ControlAX12(int id, int position, int speed) { // 1. 建立 11 Byte 控制封包陣列 byte[] dataPackage = new byte[11]; dataPackage[0] = 0xFF; // 標頭 1 (Header 1) dataPackage[1] = 0xFF; // 標頭 2 (Header 2) dataPackage[2] = (byte)id; // 目標馬達 ID (ID) dataPackage[3] = (byte)7; // 長度 (Length) - 參數位元組 + 3 dataPackage[4] = 0x03; // 指令 (Instruction) - 0x03 為 Write 指令 dataPackage[5] = (byte)30; // 起始地址 (Address) - 30 對應 Goal Position // 2. 將 16 位元資料拆分為高低位元組並填入參數 dataPackage[6] = (byte)(position & 0xFF); // Position Low Byte dataPackage[7] = (byte)(position >> 8); // Position High Byte dataPackage[8] = (byte)(speed & 0xFF); // Speed Low Byte dataPackage[9] = (byte)(speed >> 8); // Speed High Byte // 3. 計算校驗和 (Checksum) byte checksumSum = 0; for (int i = 2; i < 10; i++) { checksumSum += dataPackage[i]; } dataPackage[10] = (byte)(0xFF - checksumSum); // 存入 Checksum // 4. 經由 SerialPort 將封包陣列發送至硬體 try { serialPort1.Write(dataPackage, 0, dataPackage.Length); System.Threading.Thread.Sleep(1); // 序列穩定延遲 1ms } catch (Exception ex) { MessageBox.Show($"發送封包時出錯。\r\n{ex.Message}", "Error", MessageBoxButtons.OK, MessageBoxIcon.Error); } }
