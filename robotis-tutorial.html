一、RoboPlus Manager（硬體偵錯與設定）

控制器的硬體連線、馬達 ID 分配與運轉模式校準

在開始編寫動作邏輯前，我們必須先使用 RoboPlus Manager 與控制器進行通訊，檢測目前線路上的馬達狀態、ID 是否衝突，並進行初次轉向與模式測試。

1 實機硬體連線配置

1. 準備一台 CM-530 控制器，以及至少一顆 AX-12(A) 序列智慧馬達。
2. 使用專用的 3-Pin 傳輸線 連接控制器與馬達（馬達可採用序列式 Daisy-chain 串接）。
3. 使用 USB 傳輸線將 CM-530 連接至電腦，接上 9V ~ 12V 電源，並將控制器上的電源切換開關開啟。

▲ 實驗室常用的 CM-530 控制器

2 軟體啟動與通訊序列埠連線

1. 開啟電腦上的 RoboPlus 主程式，在中間的標籤頁選取「BIOLOID」，並點擊 「RoboPlus Manager」 啟動連線視窗。

▲ RoboPlus 套件整合開發環境

2. 在 RoboPlus Manager 視窗左上角的下拉選單中選取控制器的 COM Port。你可以在 Windows 的「裝置管理員 ➔ 連接埠 (COM 和 LPT)」中找到 COM 編號，或直接點選 「Auto Search」 讓程式自動偵測。
3. 確認 COM 設定正確後，按下旁邊的 「Connect」 開始通訊。

▲ 通訊埠連線選單

3 取得馬達清單與模式測試

連線成功後，左側會列出當前 CM-530 串接到的所有裝置資訊。對目標馬達點擊後即可進行設定。

▲ 連線成功之馬達清單

AX-12(A) 智慧序列馬達具有兩種關鍵控制模式：

JOINT 關節模式
適合機器人關節

角度範圍：0° ~ 300°
對應控制數值：0 ~ 1023
控制中心點（150°）：512
⚠️ 機械手臂組裝前，務必先將馬達轉至 512 物理中心點，否則會跑位！

WHEEL 輪子模式
適合車輪與螺旋槳

支援 360° 連續無間斷旋轉
轉向控制：CW (順時針) 或 CCW (逆時針)
速度控制範圍：0 ~ 1023 (0代表停止轉動，1023代表全速運轉)
適合用於動力行進輪。

🚨 連線釋放原則： 使用完 RoboPlus Manager 進行馬達校正與 ID 檢視後，請務必點擊 Disconnect 進行斷線！否則 CM-530 的序列通訊埠會被此視窗持續佔用，導致接下來的 Task 程式燒錄或 C# 連線出現「通訊埠拒絕存取」的錯誤。

二、RoboPlus Task（結構化指令程式撰寫）

如何透過樹狀結構邏輯，不寫複雜 C/C++ 即能快速燒錄邏輯

與一般嵌入式系統需手寫程式碼不同，Robotis 提供圖形與指令混合的 RoboPlus Task。我們只需要透過「雙擊空行 ➔ 選擇語法塊 ➔ 設定暫存器數值」即能自動編譯成燒錄檔。

1

建立專案與控制器綁定

在 RoboPlus 標籤頁開啟 RoboPlus Task。點選任意一行，在跳出的視窗中將 Firmware Version 選擇為 1.0，再於下方控制器中選擇 「CM-530」 以綁定對應的暫存器腳位。

2

理解核心語法結構與功能

雙擊空白行時會跳出指令清單。在開發機械手臂時，最基礎且必須包含的框架如下：

START PROGRAM

程式的核心進入點。如同 C 語言的 main()，所有指令必須置於其下方。

ENDLESS LOOP

無限循環區。所有即時監控、感測器讀取與狀態更新動作，均需放置於此循環中。

? = ? (指定狀態)

變數或暫存器設定。例如：將馬達 1 號 (ID 1) 的 Goal Position 設定為指定數值。

WAIT WHILE (延遲等待)

搭配內建的 Timer 作為 Delay。設定 Timer 值後，程式會暫停並等待計時結束才繼續下一動作。

3

範例程式碼實作：按鈕控制馬達

以下是一個經典的控制案例：讀取 CM-530 上的方向按鈕，以不同旋轉方向和速度來驅動馬達。

🎯 程式邏輯設定：

  • 當按下控制器上的 「U（上）」 鍵 ➔ 驅動 ID 1 馬達以 512 速度 順時針 (CW) 旋轉。
  • 當按下控制器上的 「D（下）」 鍵 ➔ 驅動 ID 1 馬達以 512 速度 逆時針 (CCW) 旋轉。
  • 其餘情況（未按按鍵） ➔ 馬達停止旋轉 (速度設為 0)。
▲ 按鍵條件控制範例
▲ 定時器 (Timer) 延遲寫法
🚀 燒錄與實機運行程序
  1. 完成語法撰寫後，點選上方工具列的 紅色勾勾 (✔️) 進行語法靜態檢測。
  2. 點擊 綠色向下箭頭 (⬇️) 燒錄 G-Code 直至燒錄完成。
  3. 控制器程序啟動：
    • 連續按壓 CM-530 上的 「MODE」 按鈕，切換模式直到左下角的 「PLAY」指示燈亮起紅燈。
    • 按下右側的 「START」 啟動，馬達即會進入程式設定邏輯，此時按壓 U/D 按鈕即可實現實機運轉控制！

🛠️ 實驗室常見故障排除：解決 UNKNOWN 馬達錯誤

控制器通訊時，Manager 無法辨識馬達型號並顯示 UNKNOWN 之自我修復指南

這是實驗室學弟妹最常遇到的硬體故障！當馬達的韌體混亂，或鮑率 (Baud Rate) 與 CM-530 預設不相符時，RoboPlus Manager 會在連線清單中將馬達標記為黃色的 UNKNOWN (未知裝置)，這會導致 Task 程式與 SDK 無法讀取任何暂存器值。

💡 UNKNOWN 馬達韌體修復 SOP：

  1. 關閉控制器電源，拔掉所有其他無關的馬達線路，只保留該顆故障馬達單獨連接 CM-530。
  2. 開啟 RoboPlus Manager，點選上方工具列的 「Firmware Recovery (韌體恢復)」 工具。
  3. 照著畫面上的精靈指示，拔除電源插頭後再重新接上，強制控制器發送特殊的重設引導廣播。
  4. 選取你的馬達型號 (AX-12A)，讓軟體自動為馬達重新刷寫全新原廠韌體，並重置 ID 至 1 號、鮑率還原為預設的 1 Mbps。

🏆 階段驗收：驗收-3 控制機械手臂

驗收前請先與負責學長姐預約實機操作評分時間

利用 RoboPlus 軟硬體套件控制你先前在 3D 建模與設備加工階段中，自行組裝與製作完成的實體機械手臂（需帶有夾爪或雙節連桿機構）。

🎯 驗收合格基準：

  • 基本連線 (20%)：能使用 RoboPlus Manager 正確讀取所有機械手臂上所安裝之 AX-12(A) 馬達 ID 與狀態。
  • 關節初始化 (30%)：程式啟動時，機械手臂的所有關節必須自動回歸預設的初始站立姿態（對應 Goal Position 各項暫存器位置設定）。
  • 基本動作展示 (50%)：新進人員撰寫 RoboPlus Task 條件程式，按壓 CM-530 上的方向控制按鈕，能展示機械手臂做出關節彎曲、夾爪開合或抓取物體的控制動作。
🔩 螺絲歸類與分類記錄（非常重要）：

在組裝與後續拆解你的手臂零件時，必須將你所用到的所有螺絲規格（例如 M2 × 10mm、M3 × 8mm 等）以列表形式記錄在你的筆記或報告中，以便在拆解復歸後，能夠將所有螺絲按照正確的長度規格分類放回零件盒中。未確實分類將直接取消驗收資格！
